Unit 6.1: Introduction to Databases

▼
  • 1 Database Fundamentals

    • What is a Database?

    A database is a collection of information that is organized in a central location so that it can be accessed, managed and updated. Databases are a collection of tables with data stored in rows and columns.

    Database structure with tables, records, and fields

    Key Database Concepts:

    • Table: A collection of data presented in rows and columns
    • Record: A row in a database table containing information about something
    • Field: A column in a database table representing an item type
    • Primary Key: A unique identifier for each record in the database
    Database Evolution: The first databases were implemented in the 1960s and 1970s using networked or flat file models. The relational database model was introduced in the late 1970s as a more flexible approach.

    Database Management Systems (DBMS)

    A Database Management System (DBMS) is computer software used to create and manage databases. Common DBMS applications include:

    Microsoft Access

    Desktop database system for small to medium applications

    MySQL

    Open-source relational database management system

    Oracle

    Enterprise-level database management system

    SQL Server

    Microsoft's enterprise database management system

  • 2 Database Structure & Design

    • Database Design Principles

    Proper database design is crucial for creating efficient, maintainable databases.

    Database design with relationships between tables

    Database Design Steps:

    1. Identify the purpose of the database
    2. Collect and organize the required information
    3. Divide information into tables
    4. Create fields (columns) for each table
    5. Specify primary keys for each table
    6. Set up relationships between tables
    Avoid Data Redundancy: Duplicate information increases the likelihood of data inconsistency. Always design databases to minimize redundancy.

    Database Administrator Role

    The Database Administrator (DBA) is responsible for:

    • Database design and implementation
    • Security and access control
    • Backup and recovery procedures
    • Performance monitoring and optimization
    • Troubleshooting database issues
  • 3 Data Types & Field Properties

    • Understanding Data Types

    Each field in a database table must have a data type that defines the kind of data it can store.

    Data Type Description Example
    Short Text Stores alphanumeric data (names, titles, etc.) John Smith
    Long Text Stores large amounts of alphanumeric data Product descriptions, comments
    Number Stores numeric data 25, 150.75
    Date/Time Stores dates and times 12/05/2023, 14:30
    Currency Stores monetary data with 4 decimal places $25.99, €150.50
    AutoNumber Unique value generated by Access for each record 1, 2, 3, ...
    Yes/No Boolean (true/false) data Yes, No
    Attachment Attach files like pictures, documents photo.jpg, document.pdf
    Calculated Expression using data from other fields Price * Quantity

    Field Properties

    Field properties control the behavior and characteristics of fields in a table:

    Field Size

    Sets the maximum size for text fields or the type for number fields

    Format

    Defines how data is displayed

    Input Mask

    Provides a pattern for data entry

    Default Value

    Automatically enters a value in new records

    Validation Rule

    Limits what can be entered in the field

    Required

    Specifies whether a value is required

Unit 6.2: Working with Tables & Relationships

▼
  • 1 Creating Tables in Access

    • Microsoft Access Interface

    When you start Access, you'll see the Backstage view where you can create new databases or open existing ones.

    Title Bar: Shows the name of the open database
    Ribbon: Collection of tabbed toolbars (Home, Create, External Data, Database Tools)
    Navigation Pane: Contains all database objects organized by type
    Workspace: Area where you work with database objects

    Creating a New Database

    To create a new database in Access:

    1. Click File → New
    2. Select Blank Database
    3. Enter a file name and choose a location
    4. Click Create
    Creating a new table in Design View

    Table Views

    Access provides different views for working with tables:

    Design View

    Used to create and modify table structure (fields, data types, properties)

    Datasheet View

    Used to enter, edit, and view table records

    Pro Tip: Always design your table structure in Design View before entering data in Datasheet View.
  • 2 Table Relationships

    • Understanding Table Keys

    Keys are used to establish relationships between tables:

    Primary Key

    A unique identifier for each record in a table. No two records can have the same primary key.

    Foreign Key

    A field in one table that corresponds to values in the primary key of another table.

    Table relationships using primary and foreign keys

    Types of Table Relationships

    There are three main types of relationships in databases:

    Relationship Type Description Example
    One-to-One (1:1) One record in Table A relates to one record in Table B Employee to EmployeeDetails
    One-to-Many (1:M) One record in Table A relates to many records in Table B Customer to Orders
    Many-to-Many (M:M) Many records in Table A relate to many records in Table B Students to Courses

    Creating Relationships in Access

    To create relationships between tables:

    1. Open the database
    2. Go to the Database Tools tab
    3. Click Relationships
    4. Add the tables you want to relate
    5. Drag the primary key field from one table to the foreign key field in another table
    6. Configure relationship options in the dialog box
    7. Click Create
    Referential Integrity: When creating relationships, consider enforcing referential integrity to maintain data consistency between related tables.

Unit 6.3: Queries, Forms & Reports

▼
  • 1 Working with Queries

    • Understanding Queries

    A query is a request for data results or for action on data. Queries allow you to:

    • Filter data based on specific criteria
    • Perform calculations on data
    • Summarize data from multiple tables
    • Combine data from related tables
    Query filtering data from multiple tables

    Types of Queries

    Access provides several types of queries:

    Select Query

    Retrieves data from one or more tables based on specified criteria

    Parameter Query

    Prompts for criteria when run

    Action Query

    Makes changes to data (update, delete, append)

    Crosstab Query

    Summarizes data in a spreadsheet-like format

    Creating Queries

    You can create queries using:

    Query Wizard

    Guides you through creating simple queries

    Query Design

    Gives you full control over query design and criteria

    Dynamic Results: When you run a query, Access creates a recordset - a dynamic set of records that reflects any changes made to the underlying tables.
  • 2 Creating Forms

    • Understanding Forms

    Forms provide a user-friendly interface for entering, editing, and viewing data. Benefits of using forms include:

    • Easier data entry than working directly with tables
    • Ability to show selected fields from a record
    • Customized layout and design
    • Data validation and input controls
    Custom form for data entry

    Form Controls

    Forms use different types of controls:

    Control Type Description Example
    Bound Control Connected to a field in a table or query Text box showing customer name
    Unbound Control Not connected to any data source Label with form title
    Calculated Control Uses an expression as data source Text box showing total price

    Creating Forms

    Access provides several ways to create forms:

    Form Tool

    Creates a simple form with one click

    Form Wizard

    Guides you through form creation with customization options

    Blank Form

    Starts with a blank form for complete design control

    Form Design

    Opens in Design view for advanced customization

  • 3 Generating Reports

    • Understanding Reports

    Reports are used to present data in a formatted, printable layout. Reports are ideal for:

    • Summarizing data for decision-making
    • Creating professional-looking documents
    • Sharing database information with others
    • Archiving data in a readable format
    Professional report with data grouping and summary

    Report Sections

    Reports are divided into sections that control where information appears:

    Report Header

    Appears once at the beginning of the report

    Page Header

    Appears at the top of every page

    Detail Section

    Contains the main body of the report

    Page Footer

    Appears at the bottom of every page

    Report Footer

    Appears once at the end of the report

    Creating Reports

    Access provides several ways to create reports:

    Report Tool

    Creates a simple tabular report with one click

    Report Wizard

    Guides you through report creation with layout options

    Blank Report

    Starts with a blank report for complete design control

    Report Design

    Opens in Design view for advanced customization

    Report Source: Reports can be based on tables or queries. Using queries as the source allows you to create reports with filtered or calculated data.

Unit 6.4: Advanced Database Features

▼
  • 1 Working with External Data

    • Importing and Exporting Data

    Access allows you to work with data from various external sources:

    Import from Excel

    Bring Excel data into Access as a new table

    Export to Excel

    Send Access data to Excel for further analysis

    Link to Excel

    Create a connection to Excel data without importing

    Text Files

    Import from or export to CSV and other text formats

    Importing data from Excel to Access

    Import vs. Link

    When working with external data, you have two options:

    Method Description When to Use
    Import Creates a copy of the data in Access When you need to modify the data in Access
    Link Creates a connection to the external data When data is maintained in the external source
    Linked Data Limitations: When you link to external data, you can view and query the data in Access, but you cannot modify the table structure.
  • 2 Working with Macros

    • Understanding Macros

    A macro is a series of commands that can be used to automate repetitive tasks in Access.

    Macros automate database tasks

    Common uses for macros:

    • Opening forms and reports automatically
    • Running queries with one click
    • Validating data entry
    • Automating import/export processes
    • Creating custom navigation systems

    Macro Security

    Because macros can contain potentially harmful code, Access provides several security settings:

    Disable all macros

    Most restrictive - no macros run

    Disable with notification

    Default setting - user can enable macros

    Enable digitally signed macros

    Only runs macros from trusted sources

    Enable all macros

    Least secure - all macros run automatically

    Security Best Practice: Use the "Disable all macros with notification" setting for the best balance of security and functionality.

    Creating Macros

    To create a macro in Access:

    1. Go to the Create tab
    2. Click Macro in the Macros & Code group
    3. Add actions from the Action Catalog
    4. Configure action arguments
    5. Save the macro
    Macro Security Warning: When you open a database containing macros, Access displays a security warning. You must enable content to run the macros.
