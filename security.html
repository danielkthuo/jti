Unit 1.1: Introduction to ICT Security

▼
  • 1 What is ICT Security?

    • Understanding ICT Security

    Information and Communication Technology (ICT) refers to technologies that provide access to data and information through the use of computers, tablets, phones and telecommunications networks.

    ICT Security protects digital assets and information

    Key Definitions:

    • Computer Security: Protection of computer systems from theft, damage, or disruption of services
    • Cybersecurity: Protection of internet-connected systems from cyberattacks
    • Information Security: Practice of preventing unauthorized access, use, disclosure, or destruction of information
    Global Village: Modern ICT has created a "global village" where people are connected by easy travel, mass media and electronic communications, making security a global concern.
  • 2 The Security Triad

    • CIA Triad: Foundation of Information Security

    The CIA triad of confidentiality, integrity, and availability is at the heart of information security. These three principles form the basis for all security measures.

    Confidentiality

    Only authorized users can access the data resources and information

    Integrity

    Only authorized users should be able to modify the data when needed

    Availability

    Data should be available to users when needed

    Confidentiality

    Integrity

    Availability

    The CIA Triad: Core principles of information security

    Extended Security Principles

    Modern security frameworks have extended the CIA triad to include additional principles:

    Authenticity

    Providing verification of the identities of users and systems

    Accountability

    Assurance by recording the identities and activities of users

    Non-repudiation

    Assuring the identities of the parties in a transaction cannot be denied

    Important: All security measures should be designed to protect one or more of these principles. A security breach typically violates at least one of these core concepts.
  • 3 Threats to Computing Environments

    • Understanding Security Threats

    Security threats can be classified based on what they target: hardware, information, or administration.

    Hardware Security: Protection of physical systems from harm, including computing devices and IoT devices
    Software Security: Protection of software against malicious attacks to ensure it functions correctly
    Personnel Security: Assurance of honesty and trustworthiness in the use of ICT resources

    Common Threats to Data Security

    Data faces numerous threats in today's digital environment:

    System Crashes

    Data lost or damaged during hardware failures, especially affecting hard disks

    Hardware Failures

    Data corrupted due to faulty disks, disk drives, or power failures

    Human Error

    Data lost by accidentally deleting or overwriting files

    Malware

    Data lost or corrupted by computer viruses and other malicious software

    Unauthorized Access

    Systems hacked by unauthorized users who delete or alter data

    Natural Disasters

    Data destroyed by natural disasters, acts of terrorism, or war

    Case Study: In 2014, Kaspersky Lab detected and neutralized over 6 billion threats, blocked 1.3 million attacks on Android devices, and identified 295,539 new malicious mobile programs.

Unit 1.2: Software-Based Threats

▼
  • 1 Types of Malicious Software

    • Understanding Malware

    Malicious software (malware) is any software that brings harm to a computer system. Malware can steal protected data, delete documents, or add unauthorized software.

    Malware comes in various forms with different harmful capabilities

    Common Types of Malware

    Virus

    Malicious program that executes itself and spreads by infecting other programs or files

    Worm

    Self-replicating malware that spreads without human interaction

    Trojan Horse

    Malicious program disguised as legitimate software

    Spyware

    Software designed to collect information without users' knowledge

    Ransomware

    Malware that encrypts data and demands payment for decryption

    Rootkit

    Malware designed to obtain administrator-level access to systems

    Mobile Threats: In 2014, 19% of Android users encountered a mobile threat, with 53% of attacks using mobile Trojans designed to steal money through SMS and banking Trojans.
  • 2 Operating System Security

    • OS Security Framework

    Operating System Security (OS Security) encompasses techniques and methods to ensure safety from threats and attacks, allowing applications to perform required tasks while preventing unauthorized interference.

    Memory Management

    Prevents programs from mixing with each other in memory

    Data Security

    Protects data from illegal use, modification or deletion

    Process Management

    Decides which tasks get CPU resources

    Device Controlling

    Controls devices using device drivers

    Authentication Methods

    Operating systems use various methods to authenticate users:

    Username/Password

    Traditional authentication using credentials

    User Card/Key

    Physical authentication using cards or key generators

    Biometrics

    Authentication using fingerprints, retina patterns, or signatures

    Program Threats

    Operating systems must protect against various program threats:

    Trojan Horse

    Traps user credentials and sends them to malicious users

    Trap Door

    Program with security holes that perform illegal actions

    Logic Bomb

    Program that misbehaves only when certain conditions are met

    Virus

    Self-replicating programs that can modify/delete files

    System Threats: Operating systems also protect against system threats like worms, port scanning, and denial of service attacks that misuse system services and network connections.

    Endpoint security protects each device connecting to the network

    Host Intrusion Prevention

    Data Loss Prevention

Unit 2.1: Securing Data and Information

  • 1 Data Protection Methods

    • Data Security Fundamentals

    Data security involves protecting data from unauthorized access, corruption, or theft throughout its lifecycle. This includes both digital and physical protection measures.

    Data security protects information throughout its lifecycle

    Data Security Techniques

    Data Encryption

    Converting data into unreadable code to prevent unauthorized access

    Data Masking

    Hiding specific data within a database to protect sensitive information

    Data Erasure

    Ensuring data is completely removed when no longer needed

    Data Backup

    Creating copies of data to restore in case of loss or corruption

    Data Security Best Practices

    Implementing strong data security requires a comprehensive approach:

    1. Identify sensitive data: Know what data needs protection
    2. Establish security policies: Create clear rules for data handling
    3. Control access: Limit data access to authorized personnel only
    4. Implement encryption: Protect data at rest and in transit
    5. Monitor and audit: Track data access and usage patterns
    6. Train employees: Ensure staff understand security protocols
    Data Classification: Not all data requires the same level of protection. Classify data based on sensitivity to apply appropriate security measures efficiently.
  • 2 Disaster Recovery Planning

    • Understanding Disaster Recovery

    Disaster recovery involves a set of policies, tools, and procedures to enable the recovery or continuation of vital technology infrastructure and systems following a natural or human-induced disaster.

    Disaster Recovery Plan Components

    Key Elements of Disaster Recovery

    Risk Assessment

    Identifying potential threats and vulnerabilities

    Business Impact Analysis

    Determining critical systems and recovery priorities

    Recovery Strategies

    Developing approaches for system recovery

    Plan Development

    Creating detailed recovery procedures

    Testing & Maintenance

    Regularly testing and updating the recovery plan

    Disaster Recovery Metrics

    Two key metrics guide disaster recovery planning:

    Recovery Time Objective (RTO)
    Maximum acceptable length of time that a system can be offline

    This determines how quickly systems must be restored after a disaster.

    Recovery Point Objective (RPO)
    Maximum acceptable amount of data loss measured in time

    This determines how frequently data must be backed up.

    Testing is Critical: A disaster recovery plan is only effective if it works when needed. Regular testing and updates are essential to ensure the plan remains viable as systems and threats evolve.
  • 3 Backup Strategies

    • Data Backup Fundamentals

    Data backup involves creating copies of data that can be restored after a data loss event. Effective backup strategies are essential for business continuity and disaster recovery.

    Regular backups protect against data loss from various threats

    Backup Types

    Full Backup

    Complete copy of all selected data

    Incremental Backup

    Only backs up data changed since last backup

    Differential Backup

    Backs up data changed since last full backup

    Mirror Backup

    Exact copy of source data without compression

    Backup Storage Options

    Choosing where to store backups is critical for security and accessibility:

    Storage Type Advantages Disadvantages
    External Hard Drives Fast, portable, inexpensive Physical damage risk, manual process
    Network Attached Storage (NAS) Centralized, automated backups Single location risk, network dependency
    Cloud Storage Off-site, accessible from anywhere Ongoing costs, internet dependency
    Tape Drives High capacity, long-term storage Slow access, specialized equipment

    Backup Best Practices

    1. Follow the 3-2-1 rule: 3 copies of data, on 2 different media, with 1 copy off-site
    2. Automate backups: Reduce human error with scheduled backups
    3. Test restores regularly: Ensure backups are working correctly
    4. Secure backup data: Encrypt backups to protect sensitive information
    5. Document procedures: Create clear instructions for backup and recovery
    Backup Verification: Creating backups is only half the solution. Regular verification that backups can be successfully restored is critical to ensure they will work when needed.

  • 1 Firewalls and Network Defense

    • Understanding Firewalls

    A firewall is a network security system that monitors and controls incoming and outgoing network traffic based on predetermined security rules. Firewalls establish a barrier between trusted internal networks and untrusted external networks.

    Firewalls act as barriers between trusted and untrusted networks

    Types of Firewalls

    Packet-Filtering Firewalls

    Examine packets and allow or block based on source/destination IP, ports, and protocols

    Stateful Inspection Firewalls

    Monitor the state of active connections and make decisions based on context

    Proxy Firewalls

    Act as intermediaries between end systems, filtering at application layer

    Next-Generation Firewalls

    Integrate additional features like intrusion prevention and application awareness

    Firewall Configuration Best Practices

    1. Default deny policy: Block all traffic by default, only allow explicitly permitted traffic
    2. Least privilege principle: Grant minimum necessary access for users and systems
    3. Regular rule review: Periodically audit and clean up firewall rules
    4. Segmentation: Use internal firewalls to segment network zones
    5. Logging and monitoring: Maintain comprehensive logs and monitor for suspicious activity
    Defense in Depth: Firewalls are just one layer of network security. Implement multiple security controls (firewalls, IDS/IPS, antivirus, etc.) to create a comprehensive defense strategy.
  • 2 Intrusion Detection and Prevention

    • Intrusion Detection Systems (IDS)

    An Intrusion Detection System (IDS) is a device or software application that monitors a network or systems for malicious activity or policy violations. Any detected activity is reported to an administrator.

    IDS/IPS Detection Methods

    Types of Intrusion Detection Systems

    Network IDS (NIDS)

    Monitors network traffic for suspicious activity

    Host IDS (HIDS)

    Monitors activity on individual devices or hosts

    Signature-Based IDS

    Detects known threats using predefined patterns

    Anomaly-Based IDS

    Detects deviations from normal behavior patterns

    Intrusion Prevention Systems (IPS)

    An Intrusion Prevention System (IPS) is an extension of IDS that can automatically take action to prevent detected threats. IPS systems can block traffic, reset connections, or drop packets.

    Feature IDS IPS
    Placement Out-of-band monitoring In-line deployment
    Action Detection and alerting Prevention and blocking
    Impact No network impact Potential latency
    Response Manual response required Automatic response

    IDS/IPS Deployment Considerations

    1. Network visibility: Ensure the system can monitor all critical network segments
    2. Performance impact: Consider the processing requirements and potential latency
    3. False positives: Tune systems to minimize false alarms while maintaining detection capability
    4. Integration: Ensure the system integrates with other security controls
    5. Maintenance: Regularly update signatures and rules to detect new threats
    IPS Challenges: Intrusion Prevention Systems can potentially block legitimate traffic if not properly configured. Thorough testing and gradual deployment are recommended to minimize business disruption.
  • 3 Virtual Private Networks (VPNs)

    • Understanding VPN Technology

    A Virtual Private Network (VPN) extends a private network across a public network, enabling users to send and receive data as if their devices were directly connected to the private network. VPNs provide secure remote access and site-to-site connectivity.

    VPNs create secure encrypted tunnels over public networks

    Types of VPNs

    Remote Access VPN

    Connects individual users to a private network from remote locations

    Site-to-Site VPN

    Connects entire networks to each other over the internet

    SSL VPN

    Uses SSL/TLS protocol to provide secure remote access via web browsers

    IPsec VPN

    Uses IPsec protocol suite to secure IP communications

    VPN Protocols and Encryption

    VPNs use various protocols to establish secure connections:

    Protocol Security Features Common Use Cases
    IPsec Strong encryption, authentication, integrity checking Site-to-site VPNs, enterprise remote access
    OpenVPN Open-source, highly configurable, strong security Remote access, cross-platform compatibility
    WireGuard Modern, fast, simple cryptography Performance-sensitive applications
    SSL/TLS Web-based, no client software needed Clientless remote access, web applications

    VPN Security Best Practices

    1. Strong authentication: Use multi-factor authentication for VPN access
    2. Encryption strength: Implement strong encryption algorithms
    3. Access controls: Restrict VPN access based on user roles and needs
    4. Network segmentation: Limit access to specific network segments via VPN
    5. Monitoring and logging: Monitor VPN connections for suspicious activity
    6. Regular updates: Keep VPN software and firmware updated
    Zero Trust Network Access (ZTNA): Modern security approaches are moving beyond traditional VPNs to Zero Trust models that verify every access request regardless of network location, providing more granular security controls.
