The Four Pillars of Computational Thinking
üß©
Decomposition

Breaking big problems into smaller, manageable parts

Example: "Make a sandwich" becomes:

  • Get bread
  • Get peanut butter
  • Spread peanut butter
  • Put pieces together
üîç
Pattern Recognition

Looking for similarities and trends

Example: In sequence 2, 4, 6, 8...

  • Pattern: Add 2 each time
  • Next number: 10
  • All even numbers
üéØ
Abstraction

Focusing on important information, ignoring irrelevant details

Example: For a map:

  • Include: Roads, names
  • Ignore: Tree colors, house details
üìù
Algorithm Design

Creating step-by-step solutions

Example: A recipe

  • Clear instructions
  • Specific order
  • Precise measurements

Abstraction: What Matters vs What Doesn't
Too Much Detail
  • Every single tree
  • Color of every house
  • Individual parked cars
  • Exact shapes of clouds
  • Every streetlight
Essential Information
  • Main roads and highways
  • Important landmarks
  • Street names
  • Direction (North/South)
  • Scale and distance
Teaching Analogy: A restaurant menu abstracts away all the kitchen details - you just see the final dishes and prices, not every cooking step.
Algorithm Design: Precision Matters!
Vague Instructions
Make a peanut butter sandwich

Problems: No specifics, assumes knowledge

Result: Confusion, wrong outcome

Precise Algorithm
Take two slices of bread
Open peanut butter jar
Pick up knife by handle
Scoop 2 tablespoons peanut butter
Spread evenly on one bread slice
Place second slice on top

Benefits: Clear, repeatable, successful

Robot Teacher Game (60 mins)
ü§ñ
The instructor becomes a robot that only understands precise commands!

Robot Vocabulary: "Move arm forward 10cm", "Turn hand 90 degrees", "Lower chalk", "Raise chalk"

"We Do" - Class vs Robot:
Class gives instructions to make robot draw a square on board
Robot follows instructions LITERALLY (creates comedy from imprecision)
Students learn need for specificity through failure
"You Do" - Pair Programming:
Students pair up: one programmer, one robot
Programmer writes algorithm for robot to stack 3 cups
Swap roles and try different task
Teaching Strategy: The robot game is intentionally frustrating and funny. Students remember the importance of precision because they experience the consequences of vague instructions.

Common Challenges & Solutions

Issue: Blocks won't snap together

Solution: Make sure you're dragging from the bump on top/bottom of blocks

Issue: Sprite disappears off screen

Solution: Use "if on edge, bounce" or set x/y positions

Issue: Animation too fast/slow

Solution: Adjust number of steps in "move" block or use "wait" blocks

Issue: Can't find specific block

Solution: Check correct color category, use search feature in blocks palette

Issue: Multiple things happening at wrong times

Solution: Use "wait" blocks to control timing and sequence

Scratch Best Practices

Start Simple: Get basic animation working before adding complexity

Test Often: Click green flag frequently to see changes

Use Comments: Add notes to explain what your code does

Experiment Freely: You can't break anything - undo is always available

Share and Remix: Learn from others' projects in the Scratch community

Module 7 Summary & Transition

Key Takeaways:

  • Computational thinking breaks complex problems into manageable parts using decomposition, pattern recognition, abstraction, and algorithms
  • Precise instructions are essential for computers (and robots!) to understand what to do
  • Scratch provides a visual, beginner-friendly introduction to programming concepts
  • Blocks represent code concepts like events, loops, and sequences
  • Programming involves both logical thinking and creative expression
  • Debugging and experimentation are natural parts of the coding process

Looking Ahead to Module 8: "Now that you understand computational thinking and basic coding, we'll bring all your digital skills together in a capstone project that showcases everything you've learned!"
